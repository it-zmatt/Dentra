# DEV_0006 — Build NavSidebar component with icon navigation and permission filtering

**Parent feature:** [FEAT_0002](../FEAT_0002.md)
**Status:** Open  *(Open → Dev → Dev Review → Testing → Done)*

## Description

Replace the empty `<nav>` stub in `App.tsx` with a real `NavSidebar` component. The sidebar is a narrow vertical strip of icon buttons — one per module — plus a logout button at the bottom. The active page icon is visually highlighted. Icons for modules the current user lacks permission to access are hidden (non-admin) or shown disabled (admin, so they can still navigate to configure permissions). The sidebar is RTL-aware: it renders on the right edge of the screen and uses `border-e` / `border-s` correctly when `dir="rtl"`.

## Scope

- **Files to create:** `src/components/common/NavSidebar.tsx`
- **Files to touch:** `src/App.tsx` — replace the `<nav>` stub with `<NavSidebar currentPage={...} onNavigate={...} />`
- **Boundary:** This DEV does NOT implement any page content — pages remain stubs. It does NOT add logout logic to the auth store — that is DEV_0004. Sidebar is icon-only in v1; a label tooltip on hover is acceptable but a label-expanded mode is out of scope.

## Requirements / Acceptance criteria

- [ ] `NavSidebar` accepts `currentPage: Page` and `onNavigate: (page: Page) => void` props
- [ ] Renders one icon button per module in this order: Patients, Calendar, Doctors, Lab Work, Expenses, Statistics, Settings
- [ ] Uses FluentUI icons from `@fluentui/react-icons` — one distinct icon per module (e.g. `PeopleRegular`, `CalendarRegular`, `PersonRegular`, `BeakerRegular`, `MoneyRegular`, `DataBarVerticalRegular`, `SettingsRegular`)
- [ ] The button corresponding to `currentPage` has a distinct active visual state (background highlight, accent color, or left/right accent bar — consistent with FluentUI design)
- [ ] A logout `Button` icon (e.g. `ArrowExitRegular`) is pinned to the bottom of the sidebar
- [ ] Clicking the logout button calls `useAuth().logout()` — no confirmation dialog needed
- [ ] Permission filtering: reads `globalSettings.permissions` (a 6-element boolean array covering Patients[0], Calendar[1], Doctors[2], LabWork[3], Expenses[4], Stats[5]); Settings is always visible; non-admin users have the icon hidden for any module where their permission is `false`; admin users see all icons regardless
- [ ] RTL layout: when `dir="rtl"`, the sidebar appears on the right side of the viewport and the accent bar for the active item appears on the left edge of the button (innermost edge); Tailwind `border-e` / `border-s` classes used correctly
- [ ] Tooltip on hover shows the module name using the i18n key for that module (e.g. `nav.patients`, `nav.calendar`)
- [ ] No hardcoded label text — all module names via i18n
- [ ] Add the 6 `nav.*` i18n keys to `ar.ts`, `fr.ts`, `en.ts`

## Testing Plan

### Input
- Render `<NavSidebar currentPage="patients" onNavigate={fn} />` with auth store populated as non-admin, `permissions = [true, true, false, true, true, true]`
- Render same with `isAdmin = true` and same permissions array
- Click each icon button
- Click the logout button
- Switch locale to `ar`

### Output
- Non-admin with `permissions[2] = false`: Doctors icon is absent from sidebar
- Admin with same permissions: Doctors icon is present (admin sees all)
- Active icon for `currentPage="patients"` is visually distinct
- Clicking another icon calls `onNavigate` with the correct page key
- Logout button calls `logout()` from auth store
- `dir="rtl"`: sidebar renders on the right, active accent on correct edge

### Test cases
1. **All permissions true:** All 7 icons visible (6 modules + Settings), correct order
2. **Permissions[3] false, non-admin:** Lab Work icon hidden; all others visible
3. **Permissions[3] false, admin:** Lab Work icon visible and enabled (admin bypass)
4. **Active state:** Navigating to Calendar highlights only the Calendar icon
5. **RTL layout:** Sidebar on right, accent bar on left edge of active icon, tooltips readable
6. **Logout:** Clicking logout icon calls auth store logout; no navigation happens inside this component
