# DEV_0003 — Wire Zustand store + i18n strings (ar/fr/en) for backup flow

**Parent feature:** [FEAT_0001](../features/FEAT_0001.md)
**Status:** Open  *(Open → Dev → Dev Review → Testing → Done)*

## Description

Create or update the Zustand store for backup operations to hold loading state and result messages. Add i18n translation strings for Arabic (ar), French (fr), and English (en) that cover all UI strings in the backup export flow: button label, loading message, success message, and error messages. All strings in the backup UI must be pulled from i18n keys (no hardcoded strings except in tests).

## Scope
- **Files likely touched:** `src/store/backupStore.ts` (new or existing), `src/i18n/ar.ts`, `src/i18n/fr.ts`, `src/i18n/en.ts`, `src/hooks/useBackup.ts` (if not already using store)
- **Boundary:** This DEV does NOT handle:
  - Restore-related strings (will be added in a separate FEAT)
  - Any UI rendering (that's DEV_0002)
  - The Rust backend (that's DEV_0001)

## Requirements / Acceptance criteria
- [ ] Zustand store `backupStore` created with state: `{ isLoading: boolean, successMessage: string | null, errorMessage: string | null }`
- [ ] Store actions: `setLoading(bool)`, `setSuccessMessage(msg: string | null)`, `setErrorMessage(msg: string | null)`, `clearMessages()`
- [ ] Store is used in `useBackup` hook to manage UI state during the export operation
- [ ] All user-facing strings in the backup UI are keyed in i18n:
  - Button label: `backup.export_button` (Arabic: "تصدير النسخة الاحتياطية", French: "Exporter la sauvegarde", English: "Export Backup")
  - Loading message: `backup.exporting` (Arabic: "جاري إنشاء النسخة الاحتياطية...", French: "Création de la sauvegarde...", English: "Creating backup...")
  - Success message template: `backup.export_success` with placeholder `{path}` (Arabic: "تم حفظ النسخة الاحتياطية على: {path}", French: "Sauvegarde enregistrée sur: {path}", English: "Backup saved to: {path}")
  - Error message (permissions): `backup.error_permissions` (Arabic: "لا يمكن الكتابة إلى المجلد المحدد...", French: "Impossible d'écrire dans le dossier sélectionné...", English: "Cannot write to selected folder...")
  - Error message (disk full): `backup.error_disk_full` (Arabic: "لا توجد مساحة كافية على القرص...", French: "Espace disque insuffisant...", English: "Insufficient disk space...")
- [ ] Strings follow the project's i18n pattern: use `useTranslation()` hook in components, pass i18n key to `t()` function
- [ ] Arabic strings are written left-to-right in the file (standard practice) but will display RTL in the UI due to RTL layout rules
- [ ] French strings use Algerian French spelling conventions where applicable (e.g., "sauvegarde" not "backup")
- [ ] All translation keys follow snake_case convention (`backup.export_button`, not `backup.exportButton`)

## Testing Plan

### Input
- Set UI language to Arabic, navigate to Settings → Backups
- Set UI language to French, navigate to Settings → Backups
- Set UI language to English, navigate to Settings → Backups
- Trigger backup export and observe all messages (loading, success, error)

### Output
- All UI text appears in the selected language
- Arabic text displays in RTL layout
- French and English text displays in LTR layout
- Success message shows the correct backup file path
- Error messages are clear and actionable in all three languages

### Test cases
1. **Arabic UI:** All strings display correctly in Arabic with RTL layout
2. **French UI:** All strings display correctly in French with LTR layout
3. **English UI:** All strings display correctly in English with LTR layout
4. **Dynamic strings:** Success message correctly interpolates the backup file path in all languages
5. **Store state:** Loading state transitions correctly during the operation (isLoading: false → true → false)
