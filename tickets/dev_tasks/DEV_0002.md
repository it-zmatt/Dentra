# DEV_0002 — Add Backup section UI in Settings panel with folder picker

**Parent feature:** [FEAT_0001](../features/FEAT_0001.md)
**Status:** Open  *(Open → Dev → Dev Review → Testing → Done)*

## Description

Add a new Backups section to the Settings panel (admin-only). The section includes a button labeled "Export Backup" that opens the Tauri folder picker, collects the destination folder path from the user, calls the `export_backup` Rust command, and displays a success or error message. The UI must be responsive — show a loading overlay while the operation is in progress, and handle errors gracefully with clear messaging.

## Scope
- **Files likely touched:** `src/pages/SettingsPage.tsx`, `src/components/settings/BackupSection.tsx` (new), `src/hooks/useBackup.ts` (new or update existing if it exists), `src/services/backup.ts` (new wrapper around invoke), `tauri.conf.json` (declare `dialog` plugin if not already present)
- **Boundary:** This DEV does NOT handle:
  - Restore from backup (separate DEV)
  - i18n strings (that's DEV_0003)
  - The Rust command implementation itself (that's DEV_0001)

## Requirements / Acceptance criteria
- [ ] New Backups section added to Settings page, visible only to admin users
- [ ] "Export Backup" button is prominent and uses FluentUI Button component with appropriate styling
- [ ] Clicking the button opens Tauri's native folder picker dialog (not a file picker — must be directory picker)
- [ ] After user selects a folder, folder path is passed to `export_backup` Rust command via `invoke()`
- [ ] During the async operation, a loading overlay is shown with message "Creating backup..." (to be i18n'd in DEV_0003)
- [ ] On success, a confirmation message is shown with the full path: "Backup saved to [path]/dental_backup_YYYY-MM-DD.zip" — message is dismissible (user can click OK or close)
- [ ] On failure, error message is shown to the user with clear language (e.g., "Cannot write to selected folder. Check permissions and try again.")
- [ ] If the user cancels the folder picker, no command is invoked and no error is shown
- [ ] Frontend never calls `invoke()` directly from the component — use `src/services/backup.ts` wrapper function
- [ ] Hook follows the pattern in `.claude/CLAUDE.md`: `useBackup` hook calls service, service calls invoke, component calls hook
- [ ] Folder picker capability is declared in `tauri.conf.json` under `plugins.dialog`
- [ ] Uses `@fluentui/react-components` for all UI primitives (Button, loading spinner, notification)

## Testing Plan

### Input
- User is logged in as admin
- Navigate to Settings → Backups section
- Click "Export Backup" button
- Native folder picker opens
- User selects a valid writable folder (e.g., Desktop)

### Output
- Loading overlay appears for ~2 seconds (while zip is created)
- Success message appears: "Backup saved to C:\Users\...\Desktop\dental_backup_2026-04-26.zip"
- File exists at that path and is a valid zip archive
- Message can be dismissed by clicking OK or the close icon

### Test cases
1. **Happy path:** Admin selects a writable folder, backup succeeds, confirmation message shown
2. **Folder picker cancelled:** User opens picker and clicks Cancel; no action taken, UI returns to normal
3. **Folder picker denied:** User clicks Cancel button; no async operation started, no error shown
4. **Permission error:** Admin selects a read-only folder; error message appears: "Cannot write to selected folder. Check permissions and try again."
5. **Disk full error:** Admin selects a destination with insufficient space; error message appears: "Insufficient disk space for backup."
